LRU-K Алгоритм вытеснения кэша

Общее описание

Данный проект представляет собой реализацию алгоритма вытеснения кэша LRU-K (Least Recently Used with K-th reference) на языке C. 
Алгоритм предназначен для эффективного управления кэш-памятью в системах, где необходимо быстро принимать решения о том, 
какие данные следует сохранять, а какие — вытеснять при заполнении кэша.

LRU-K является улучшенной версией классического LRU (LRU-1) алгоритма. 

В отличие от стандартного LRU, который учитывает только самое последнее обращение к элементу, 
LRU-K анализирует K последних обращений для принятия более обоснованных решений о вытеснении. 
Это делает алгоритм более устойчивым к сканированию (scan resistance) — однократные обращения к большому объёму данных 
не вытесняют часто используемые элементы.

Теоретическая часть

1. Основные понятия кэширования

Кэширование — это техника хранения копий данных в высокоскоростной памяти для ускорения доступа. 
Кэш имеет ограниченный размер, поэтому необходимо выбирать, какие данные хранить, а какие удалять при заполнении.

Ключевые термины:

- Кэш-промах (Cache Miss) — ситуация, когда запрошенные данные отсутствуют в кэше, и их необходимо загрузить из более медленной памяти
- Кэш-попадание (Cache Hit) — ситуация, когда запрошенные данные уже находятся в кэше
- Hit Rate — отношение количества попаданий к общему числу обращений
- Вытеснение (Eviction) — удаление данных из кэша для освобождения места

2. Алгоритм LRU (Least Recently Used)

Классический LRU алгоритм хранит элементы в порядке их последнего использования. 
При вытеснении удаляется элемент, к которому дольше всего не было обращений.

Принцип работы:

При каждом обращении к элементу он перемещается в конец списка (становится "самым новым")

При вытеснении удаляется элемент из начала списка (самый старый)

Преимущества:
- Простота реализации
- Эффективен для рабочих нагрузок с локальностью обращений

Недостатки:
- Уязвим к сканированию (однократные обращения вытесняют полезные данные)
- Не учитывает частоту обращений

3. Алгоритм LRU-K

LRU-K расширяет классический LRU, учитывая историю K последних обращений к каждому элементу. 
Элемент считается "ценным" только после того, как к нему обратились как минимум K раз.

Принцип работы:
- История обращений: для каждого элемента хранится время последних K обращений
- Холодные элементы: элементы с менее чем K обращениями считаются "холодными" и вытесняются в первую очередь
- Горячие элементы: элементы с K и более обращениями считаются "горячими"

Вытеснение:
- Сначала удаляются "холодные" элементы (с наименьшим количеством обращений)
- Если все элементы "горячие", удаляется элемент с самым старым K-м обращением

Преимущества LRU-K:
- Устойчив к сканированию (однократные обращения не вытесняют часто используемые данные)
- Лучше предсказывает будущие обращения, чем LRU-1
- Эффективен для рабочих нагрузок с разной частотой доступа
- Хорошо работает с паттернами доступа, где важна частота, а не только недавность

Недостатки:
- Требует больше памяти для хранения истории обращений
- Более сложная реализация

4. Алгоритм LRU-2 (K=2)

Наиболее распространённая версия LRU-K — LRU-2, где K=2. 
Это означает, что элемент считается "горячим" только после двух обращений к нему.

Почему K=2 наиболее популярен:
- Баланс между памятью и точностью
- Достаточно для защиты от однократных сканирований
- Меньшие накладные расходы, чем при K>2

Ключевые структуры данных

1. Узел истории обращений (AccessNode). Хранит одно обращение к элементу. История содержит до K последних обращений.
2. Элемент кэша (CacheItem). Хранит данные, историю обращений и используется в структурах для быстрого доступа.
3. LRU-K Кэш (LRUKCache)

Компоненты реализации

1. Хеш-таблица (Hash Table)
Назначение: быстрый поиск элемента по ключу (O(1)).

Реализация:

- Использует хеш-функцию FNV-1a для преобразования строкового ключа в индекс
- Линейное пробирование для разрешения коллизий
- Размер таблицы в 2 раза больше ёмкости кэша для уменьшения коллизий

2. LRU-список (LRU List)

Назначение: поддержание порядка обращений к элементам.

Реализация:
- Двусвязный список для O(1) вставки и удаления
- Голова — самый старый элемент
- Хвост — самый новый элемент (последнее обращение)

Операции:

lru_add_to_tail() — добавление нового элемента в конец

lru_move_to_tail() — перемещение существующего элемента в конец (при обращении)

lru_remove() — удаление элемента из списка

3. Куча (Heap)

Назначение: быстрый выбор элемента для вытеснения.

Реализация:
- Минимальная куча (min-heap) на основе массива
- Ключ сравнения — время K-го обращения
- O(log n) для операций вставки/извлечения

Принцип работы:
- Корень кучи — элемент с наименьшим временем K-го обращения
- При изменении времени K-го обращения — обновление позиции в куче
- При удалении элемента — удаление из кучи

LRU-K является мощным алгоритмом управления кэшем, который превосходит классический LRU в сценариях с неоднородным доступом и сканированием. 
Данная реализация на C обеспечивает:
- Высокую производительность благодаря хеш-таблице для O(1) поиска
- Эффективное управление памятью с использованием кучи для выбора
- Гибкость через параметр K для настройки под разные рабочие нагрузки
- Полную статистику для мониторинга эффективности кэша

Алгоритм успешно применяется в системах управления базами данных, веб-серверах, файловых системах и других областях, 
где требуется эффективное управление ограниченными ресурсами кэш-памяти.
